I'm still new to coding and computational physics especially, but, before taking this class, I had taken a few math courses that were more focused on computation, these being Applied ODE's and Numerical Linear algeba. The Applied ODE class was mostly plotting solutions and wasn't the most coding intensive, but, for Numerical Linear algebra, it definitely was. 

The main themes of Numerical Linear algebra was taking concepts from linear algebra and implemeneting them computationally. Instead of solving large systems by hand, we can write code that allows us to solve these large systems, computing things like eigenvalues, eigenvectors, and singular value decomposition of matrices. Examples of this are the Rayleigh quotient, which can be used to approximate the eigenvalues of a given matrix for which they cannot be solved by hand. Another method is the power method, which can be used to approximate the eigenvectors of a given matrix, as well as its eigenvalues. This helped highlight the idea that computation allows us to extend mathematical techniques to very large systems that would be hard to solve otherwise. 

An example of this computational theme is for Einstein's field equations, which are a set of nonlinear differential equations which cannot be solved using regular methods for solving differential equations. The only way we can solve these equations are by applying numerical solutions and computational models to understand how the space-time coordinates are curved. 

Applying the same logic to computational physics that we did with math, we can take physical questions and apply computational methods, allowing us to better understand the given models.
